## 🎯 Why QueryPilot Was Built

QueryPilot follows the same philosophy as CBT Pro, the portfolio site, and the HMG Concepts site:

1. **Solve a real problem you have watched up close.** Adewale teaches data analysis and CS to students who cannot afford a SQL textbook, let alone an AI subscription. They need a free, offline tool that explains SQL in plain language.
2. **Use AI as a collaborator, not a crutch.** The tool itself contains no AI API — it uses rule-based engines — but the developer leveraged AI to design, refactor, and document the codebase.
3. **Ship on the smallest possible footprint.** Single-file (v6) or 5-file PWA (v7). No backend, no API costs, no recurring fees. Anyone with a browser can use it forever, even offline.
4. **Build for the Nigerian learner first — and everyone benefits.** What works for a student on an entry-level Android tablet in Ondo State also works on a MacBook in San Francisco.

## 📜 The Working Method (for builders curious about the *how*)

If you're a developer wondering how a "non-web-developer" ships polished web apps:

1. **Start with a real, observed problem.** Not a tutorial topic. A real thing you watched fail.
2. **State the problem in plain language.** If you can't write it down for a non-technical reader in three sentences, you don't understand it yet.
3. **Sketch the solution on paper / in notes.** Not in code. Decide what the user must be able to do, in order.
4. **Use AI as a senior pair-programmer.** Ask it for ES5-compatible code, vanilla-JS-only constraints, accessibility patterns. Read every line it gives you. Reject what doesn't fit.
5. **Ship the smallest possible thing.** Single HTML file if possible. Add features only when a real user asks.
6. **Document honestly.** Including the things that didn't work. The data-leakage fix in SwiftChain is still in that notebook. That's the whole point.
7. **Stay in the classroom (or the equivalent for your field).** The problems live there. The vocabulary lives there. The validation lives there.

That's the method. It does not require a CS degree, a laptop, a budget, or a permission slip. It requires clear thinking — and most teachers already have that.
