What enterprise teams get

📁 Workspaces

Separate schemas, saved queries, templates, comments and glossary entries for different clients, teams or environments.

👑 RBAC

Viewer, Analyst, DBA and Admin roles enforce intentional permissions in the UI.

📜 Audit Log

Append-only action history for workspace changes, query operations, approvals, backups, vault actions and compliance exports.

🛡️ Governance Rules

Rule-based blockers and warnings for dangerous SQL, sensitive actions and production-risk patterns.

🔍 PII Scanner

Detects sensitive columns such as email, phone, BVN-like, names and identifiers in schemas.

✅ Approvals

Workflow for submitting high-risk queries and approving/rejecting them in a role-aware way.

🌳 Lineage

Shows how queries depend on tables and columns, supporting impact analysis and documentation.

🧹 Linter

50+ SQL-style and safety checks help standardise query quality.

🔄 Migrations

Generate migration scripts between schema versions for controlled change management.

🎲 Test Data

Generate local fake rows for demonstrations, testing and training without exposing real data.

💰 Cost Estimator

Heuristic risk and cost scoring based on joins, subqueries, wildcard scans, sorting and complexity.

🔐 Vault

Encrypted local vault stores sensitive notes locally using browser crypto where available.

Static enterprise architecture note

SQL v5 is local-first. It is suitable for education, analysis, documentation, governance rehearsal and browser-based SQL workflows. For private production databases, add a separate serverless API layer so credentials are never exposed in frontend code.
